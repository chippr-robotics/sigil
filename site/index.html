Institutional-Grade Security

Built for capital markets with uncompromising security standards

Physical Containment

Key material stored on air-gapped floppy disks. No private key ever touches a networked device.

Bounded Damage

If a disk is compromised, exposure is limited to N presignatures. Attacker cannot generate new ones.

Full Auditability

Every signature executes inside zkVM, producing cryptographic proofs of correct computation.

Expiration Controls

Presignatures expire after configurable periods, forcing periodic reconciliation and review.

Dual Authorization

Neither party can sign alone. Requires both cold shard (disk) and agent shard for any transaction.

Agent Integration

Native CLI tools for AI agents. Claude can request signatures with human-in-the-loop disk insertion.

Cryptographic Foundation

Built on proven cryptographic primitives and cutting-edge threshold signature schemes

2-of-2 Threshold ECDSA

Sigil implements secure two-party ECDSA using presignatures. The private key is split between two parties, and signatures require both shares to compute.

// Signature computation s = k-1 · (z + r · x) mod n // Where: // k = k_cold + k_agent (reconstructed nonce) // x = x_cold + x_agent (reconstructed key)

FROST Threshold Signatures

For modern blockchain networks, Sigil implements FROST (Flexible Round-Optimized Schnorr Threshold) signatures with support for multiple curve types.

// Schnorr signature (simpler than ECDSA) s = k + e · x mod n // Enables: // - Batch verification (3x faster) // - Native adaptor signatures // - True threshold signing

Zero-Knowledge Proofs

All signing operations execute inside the SP1 zkVM, producing verifiable proofs that the signature was computed correctly from valid presignature shares.

Security Properties

  • Unforgeability Single-party compromise cannot produce signatures
  • Bounded Exposure Disk theft limits attacker to N presignatures
  • Forward Secrecy Past signatures remain secure after compromise
  • Auditability zkVM proofs verify every operation

Cryptographic Assumptions

  • ECDLP hardness on secp256k1
  • Random oracle model (SHA-256)
  • SP1 zkVM soundness
  • 256-bit entropy from system RNG
